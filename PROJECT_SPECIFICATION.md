# OnlyFriends — спецификация проекта

**Версия документа:** 1.0  
**Дата:** 7 июля 2026  
**Статус продукта:** MVP / рабочая версия для локального развёртывания

---

## 1. Краткое описание

**OnlyFriends** — веб-панель управления флотом аккаунтов XVIDEOS с фоновой автоматизацией через Playwright. Система закрывает полный цикл outreach: регистрация и верификация почты, поиск целевых профилей, массовые заявки в друзья, подписки, сообщения взаимным друзьям, прогрев аккаунтов и мониторинг банов.

Интерфейс полностью на **русском языке**. Панель рассчитана на локальный запуск оператором (без встроенной авторизации в UI).

### Ключевая ценность для покупателя

- Готовая автоматизация XVIDEOS «из коробки» — не нужно писать скрипты с нуля
- Централизованное управление десятками/сотнями аккаунтов
- Очередь задач с защитой от банов (задержки, лимиты, прокси)
- Поиск целей, распределение по аккаунтам, цепочка «друзья → подписка»
- Отслеживание ошибок, капчи и статусов в одном интерфейсе

## 2. Целевое использование

| Сценарий | Описание |
|----------|----------|
| Массовая регистрация | Создание до 50 аккаунтов за раз с автогенерацией данных и temp-mail |
| Outreach | Поиск профилей → заявки в друзья → автоподписка после успеха |
| Прогрев | Имитация просмотра контента для снижения риска блокировок |
| Сообщения | Рассылка шаблонного текста взаимным друзьям |
| Операционный контроль | Дашборд, очередь задач, журнал сбоев, мониторинг банов |

## 3. Функциональные модули

### 3.1. Дашборд (`/`)

- Сводка по флоту: всего аккаунтов, активных, в капче, забаненных
- Статистика outreach: друзья, подписки, топ аккаунтов по активности
- Последние задачи в очереди
- Быстрые ссылки на проблемные аккаунты

### 3.2. Аккаунты (`/accounts`, `/accounts/[id]`)

**Создание и управление:**
- Одиночное и массовое создание (до **50** аккаунтов за операцию)
- Автогенерация женских имён, username и пароля
- Автоматическое создание временной почты (Mail.tm) для каждого аккаунта
- Привязка к группе и прокси
- Опция «сразу зарегистрировать на XVIDEOS» (job `REGISTER`)

**Операции с аккаунтом:**
- Редактирование профиля (имя, bio, аватар, email, пароль)
- Синхронизация профиля на XVIDEOS (`UPDATE_PROFILE`)
- Регистрация (`REGISTER`), верификация почты (`VERIFY_EMAIL`)
- Прогрев (`WARMUP_SCROLL`)
- Просмотр карточки аккаунта на XVIDEOS в браузере
- Синхронизация статистики друзей с сайта

**Детальная страница аккаунта:**
- Вкладки: взаимные друзья, отправленные заявки, подписки, сбои outreach
- История задач по аккаунту
- Счётчики mutual friends / sent requests

**Статусы аккаунта:**

| Статус | Значение |
|--------|----------|
| `IDLE` | Создан локально, сессия не активна |
| `ACTIVE` | Рабочая сессия, готов к автоматизации |
| `CAPTCHA` | Заблокирован капчей, нужно ручное решение |
| `BANNED` | Аккаунт забанен на XVIDEOS |
| `ERROR` | Ошибка автоматизации |

### 3.3. Рассылка / поиск (`/search`)

**Поиск профилей XVIDEOS:**
- Фильтры: пол, страна, возраст, верификация, режим списка, сортировка, ключевые слова, дата создания
- Пагинация результатов
- Чекбоксы для ручного выбора целей
- Панель выбора аккаунтов-отправителей

**Отправка выбранных:**
- Мгновенная постановка в очередь: друзья + подписка на выбранные профили
- Распределение целей между аккаунтами (один профиль — один аккаунт)
- Оценка времени выполнения

**Быстрая рассылка (модальное окно):**
- Указание количества целей (1–500, пресеты 10/25/50/100/500)
- Выбор аккаунтов-исполнителей
- Источник целей:
  - **Из поиска** — автосбор по текущим фильтрам (многофазный: строгий → расширенный → неверифицированные → новые)
  - **Из выбранных** — только отмеченные чекбоксами профили (без переключения на поиск)
- Предпросмотр списка целей перед запуском
- Цепочка: `ADD_FRIENDS` → автоматический `SUBSCRIBE` после успеха

### 3.4. Взаимные друзья / сообщения (`/friends`)

- Список взаимных друзей по всем активным аккаунтам (синхронизация с XVIDEOS)
- Редактируемый шаблон сообщения (сохраняется в настройках)
- Отправка сообщения с конкретного аккаунта конкретному другу
- Статусы: в очереди / отправлено / ошибка
- Прямая ссылка на чат XVIDEOS

**Ограничения чата (определяются сайтом):**
- Нужны взаимные друзья
- На аккаунте должен быть настроен ключ шифрования чата
- Некоторые профили используют Sheer chat (стандартный чат недоступен)
- Требуется разрешение на чат с профилем

### 3.5. Временная почта (`/mail`)

- Просмотр inbox Mail.tm (отдельный sandbox, не привязан к аккаунтам бота)
- Извлечение ссылок верификации из писем
- Используется в основном для отладки; верификация аккаунтов идёт автоматически через worker

### 3.6. Группы (`/groups`)

- Создание групп аккаунтов (название, описание, цвет)
- Фильтрация и организация флота при массовых операциях

### 3.7. Прокси (`/proxies`)

- CRUD прокси: HTTP, HTTPS, SOCKS5
- Поля: host, port, логин/пароль, страна, активность
- Тест соединения (проверка IP и доступности XVIDEOS)
- Привязка прокси к аккаунту (1:1)
- Автосоздание прокси из переменных окружения при старте

### 3.8. Задачи (`/jobs`)

- Просмотр очереди: тип, статус, аккаунт, прогресс, ошибка
- Отмена pending-задач
- Возобновление задач на паузе из-за капчи
- Открытие видимого браузера для ручного решения капчи

### 3.9. Сбои рассылки (`/issues`)

- Сводка failed/skipped операций друзья/подписка по всему флоту
- Подсказки по типу ошибки
- Фильтр: все / только ошибки / только пропущенные

## 4. Автоматизация (типы задач)

| Тип задачи | Описание | Триггер |
|------------|----------|---------|
| `REGISTER` | Регистрация на XVIDEOS, FriendlyCaptcha, сохранение cookies | Создание аккаунта с autoRegister |
| `VERIFY_EMAIL` | Запрос письма + переход по ссылке из Mail.tm | Автоматически после REGISTER |
| `UPDATE_PROFILE` | Аватар, имя, bio, пол на XVIDEOS | Редактирование аккаунта с sync |
| `ADD_FRIENDS` | Заявки в друзья по списку targets | Рассылка / быстрая рассылка |
| `SUBSCRIBE` | Подписка на профили | Цепочка после ADD_FRIENDS или вручную |
| `SEND_MESSAGE` | Сообщение в чат XVIDEOS | Страница взаимных друзей |
| `WARMUP_SCROLL` | Скролл и просмотр контента (по умолчанию 5 мин) | Кнопка прогрева на аккаунте |
| `LOGIN` | ⚠️ Объявлен в схеме, **не реализован** в worker | — |

### Цепочка outreach

```
Поиск / выбор целей
       ↓
ADD_FRIENDS (с chainSubscribe: true)
       ↓ успех
SUBSCRIBE на тех же targets (с задержкой ~45 сек)
       ↓
Синхронизация статистики друзей
       ↓
SEND_MESSAGE взаимным друзьям (опционально)
```

### Защита от дублей

- Таблица `FriendTargetClaim`: один username назначается только одному аккаунту
- Уже добавленные / с подпиской профили пропускаются навсегда

## 5. Архитектура

```
┌─────────────────────────────────────────────────────────┐
│  Next.js 15 (UI + API Routes)                           │
│  React 19 · Tailwind CSS · Prisma · SQLite              │
└───────────────────────┬─────────────────────────────────┘
                        │ POST jobs / CRUD
                        ▼
┌─────────────────────────────────────────────────────────┐
│  SQLite (dev.db) — accounts, jobs, proxies, claims    │
└───────────────────────┬─────────────────────────────────┘
                        │ poll PENDING
                        ▼
┌─────────────────────────────────────────────────────────┐
│  Worker (отдельный процесс)                            │
│  tsx src/worker/index.ts                                │
│  → claim jobs → Playwright Chromium                     │
│  → proxy + cookies + fingerprint per account            │
└─────────────────────────────────────────────────────────┘
```

### Worker

- Опрашивает БД каждые `WORKER_POLL_MS` (по умолчанию 3 сек)
- Параллельность: общая, на прокси, на outreach — настраивается
- Один аккаунт — одна активная задача
- Зависшие задачи (>15 мин RUNNING) → FAILED
- `kickJobProcessor()` — немедленный запуск цикла после постановки задачи из API

### Playwright-сессия

- Детерминированный fingerprint (viewport, timezone, locale) по accountId + стране прокси
- Anti-detect: скрытие webdriver, plugins, languages
- Сохранение и восстановление cookies между задачами
- `repairAccountCookies` / `ensureXvideosSession` — восстановление сессии после сбоев (friends/subscribe)

## 7. Инструкция по установке и запуску

### 7.1. Требования

- **Node.js** 20+ (рекомендуется LTS)
- **npm** 10+
- **macOS / Linux / Windows** (Playwright поддерживает все три)
- Стабильный интернет; для outreach — **резидентные прокси** (настоятельно рекомендуется)

### 7.2. Первичная установка

```bash
# 1. Распаковать проект и перейти в каталог
cd OnlyFriends

# 2. Установить зависимости (автоматически prisma generate)
npm install

# 3. Установить браузер Playwright (один раз)
npx playwright install chromium

# 4. Скопировать конфигурацию
cp .env.example .env

# 5. При необходимости отредактировать .env (см. раздел 8)

# 6. Создать / применить схему БД
npm run db:push

# 7. (Опционально) Заполнить прокси из env
npm run proxy:seed
```

### 7.3. Запуск в работу

Нужны **два процесса** (или один комбинированный):

```bash
# Терминал 1 — веб-панель (http://localhost:3000)
npm run dev

# Терминал 2 — фоновый worker (выполняет автоматизацию)
npm run worker
```

Или одной командой:

```bash
npm run dev:all
```

### 7.4. Дополнительные команды

| Команда | Назначение |
|---------|------------|
| `npm run build` | Production-сборка Next.js |
| `npm run start` | Запуск production-сервера |
| `npm run workers` | 4 параллельных worker-процесса |
| `npm run monitor:bans` | Периодическая проверка банов всех аккаунтов |
| `npm run proxy:test` | Тест прокси из .env |
| `npm run db:studio` | Prisma Studio — просмотр БД |

### 7.5. Решение капчи

1. Worker ставит задачу в `PAUSED_CAPTCHA`, аккаунт → `CAPTCHA`
2. Открыть `/jobs` → кнопка «Открыть браузер»
3. Решить капчу в появившемся окне Chromium
4. Нажать «Продолжить» / Resume

Для удобства можно установить `HEADLESS=false` в `.env` — браузер будет видимым.

### 7.6. Типовой рабочий процесс оператора

1. **Прокси** — добавить на `/proxies`, привязать к аккаунтам
2. **Аккаунты** — создать пачку (до 50), включить autoRegister
3. **Дождаться** REGISTER + VERIFY_EMAIL в `/jobs`
4. **Прогрев** — опционально WARMUP на новых аккаунтах
5. **Рассылка** — `/search`: фильтры → выбор аккаунтов → быстрая рассылка или отправка выбранных
6. **Мониторинг** — `/issues` для ошибок, `monitor:bans` для банов
7. **Сообщения** — синхронизировать статистику друзей → `/friends` → отправить шаблон

## 8. Конфигурация (.env)

```env
# База данных
DATABASE_URL="file:./dev.db"

# Worker
WORKER_POLL_MS=3000
WORKER_CONCURRENCY=4          # Всего параллельных задач
WORKER_PROXY_CONCURRENCY=1    # Задач на один IP прокси
WORKER_OUTREACH_CONCURRENCY=2 # Параллельных outreach-задач
WORKER_START_STAGGER_MS=1500  # Пауза между стартами задач
WORKER_STALE_JOB_MS=900000    # Таймаут зависшей задачи (15 мин)

# Защита от банов
REQUIRE_PROXY_FOR_OUTREACH=true   # Outreach без прокси запрещён
OUTREACH_DELAY_MIN_MS=5000        # Задержка между targets (мин)
OUTREACH_DELAY_MAX_MS=12000       # Задержка между targets (макс)
OUTREACH_BATCH_PAUSE_MIN_MS=15000
OUTREACH_BATCH_PAUSE_MAX_MS=35000
OUTREACH_HOURLY_CAP_PER_ACCOUNT=40  # Лимит outreach/час на аккаунт (0 = без лимита)
SUBSCRIBE_STAGGER_MS=45000        # Пауза перед подпиской после друзей

# Автосоздание прокси при старте (опционально)
PROXY_NAME="My VPN"
PROXY_HOST=""
PROXY_PORT="8080"
PROXY_TYPE="HTTP"
PROXY_USERNAME=""
PROXY_PASSWORD=""
PROXY_COUNTRY="US"
```

### Рекомендуемые настройки для продакшена

- `REQUIRE_PROXY_FOR_OUTREACH=true` — обязательно
- Отдельный прокси на каждый аккаунт (или минимум ротация)
- `WORKER_CONCURRENCY=2–4`, не завышать без тестов
- `OUTREACH_HOURLY_CAP_PER_ACCOUNT=20–40` — начать консервативно
- Запускать `monitor:bans` в фоне (cron или screen/tmux)

## 11. Известные ограничения

1. **SQLite** — однопользовательская БД; для масштаба >1000 аккаунтов рекомендуется миграция на PostgreSQL
2. **Нет авторизации UI** — панель открыта без пароля (предполагается localhost / VPN)
3. **LOGIN не реализован** — восстановление сессии через cookie repair, не через форму логина
4. **Капча** — только ручное решение, без внешнего solver-сервиса
5. **Парсинг HTML XVIDEOS** — зависит от вёрстки сайта; при обновлении сайта селекторы могут потребовать правки
6. **Сообщения** — только взаимным друзьям; нужна первичная настройка чата на XVIDEOS
7. **Подписка** — требует верифицированный email
8. **Регистрация** — только женские профили (хардкод `Woman`)
9. **Один target — один аккаунт** — глобальный claim, нельзя дублировать outreach
10. **Лимит друзей (code 11)** — при достижении лимита XVIDEOS остаток batch пропускается
11. **`outreachBatchPause`** — объявлен в конфиге, но не вызывается в коде (batch-pause не активен)
12. **Mail page** — отдельный sandbox, не связан с email аккаунтов бота

- Исходный код проекта (TypeScript / Next.js)
- Prisma-схема и миграции

- Поддержка изменений вёрстки XVIDEOS после обновлений сайта
- Гарантия обхода банов (система снижает риск, но не устраняет его полностью)

## 13. Рекомендации покупателю при внедрении

1. Начать с **5–10 аккаунтов** и тестового outreach, наблюдать `/issues`
2. Использовать **резидентные прокси** (1 IP ≈ 1–3 аккаунта)
3. Включить **прогрев** перед массовой рассылкой
4. Не превышать **hourly cap** без необходимости
5. Регулярно запускать **monitor:bans**
6. Перед сообщениями — **синхронизировать статистику друзей** и настроить чат на XVIDEOS вручную один раз
7. Делать **бэкап `dev.db`** перед массовыми операциями
8. Для production рассмотреть миграцию на PostgreSQL и добавление auth на панель
